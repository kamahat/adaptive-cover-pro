# v2.22.0-beta.7

beta.7 delivers one new feature (custom sunrise/sunset time entities) and two bug fixes against the My Position entities shipped in beta.6.

## 🎯 Should I install this?

- **If you use Sun2 or another integration that exposes elevation-defined sunrise/sunset times, and the fixed-offset sliders aren't accurate enough across seasons**, install this. Two new optional entity pickers (`Sunset Time Entity`, `Sunrise Time Entity`) let you wire any `sensor` or `input_datetime` whose state is an ISO-8601 datetime directly into the sunset window calculation.
- **If you installed beta.6 and noticed `number.<managed_cover>_my_position_value` showed `unknown` after every options save**, install this. A missing `native_value` property on the number entity caused the display to reset on every config-entry reload; beta.7 fixes it.
- **If you are on beta.6 and neither of the above applies**, this update is low-risk. No existing behavior changes outside the fixed entities.
- **If you are on v2.21.5 without any active issue**, stay there. This beta exists to close out feature work before the v2.22.0 stable release.

## 🔑 What's new in beta.7

### ✨ Features

- **Custom sunrise/sunset time entities ([#411], PR [#415], commit 27f5e5d):**

  Two new optional entity options — `Sunset Time Entity` (`CONF_SUNSET_TIME_ENTITY`) and `Sunrise Time Entity` (`CONF_SUNRISE_TIME_ENTITY`) — accept a `sensor` or `input_datetime` whose state is an ISO-8601 datetime string. When set, the entity value replaces the astral-computed boundary inside `compute_effective_default()` in `helpers.py`. Existing `Sunset Offset` / `Sunrise Offset` sliders remain additive on top of the entity value. The integration falls back to astral silently when the entity is unavailable, unknown, or unparseable.

  Typical use case: a Sun2 "sunset at -4° elevation" sensor delivers a seasonally-adjusted boundary that the static astral value cannot match.

  A new module-level helper `_read_time_entity` in `coordinator.py` reads the entity state via `get_safe_state`, parses it with `get_datetime_from_str`, and returns a naive-local datetime or `None` on any failure. The call site in the coordinator passes the result through to `compute_effective_default` as `sunset_time` / `sunrise_time` keyword arguments.

  Translations updated for DE and FR.

### 🐛 Bug Fixes

- **`my_position_value` number showed `unknown` after every options save ([#409], PR [#413], commit 0eeccb4):**

  `AdaptiveCoverMyPositionNumber` defined `async_set_native_value` (the write path) but no `native_value` property. HA's base class returned `_attr_native_value = None` by default. Because every options-form save triggers a full config-entry reload, the entity was recreated with no memory of the persisted value. The fix adds a `native_value` property that reads `CONF_MY_POSITION_VALUE` directly from `config_entry.options`, mirroring the live-read pattern the My Position button already used.

- **My Position entity names: disambiguated + Somfy "My" brand preserved ([#409], PR [#413], commit 0eeccb4):**

  EN entity names `"My Position"` and `"My Position Value"` were visually identical enough to cause confusion in dashboards. Renamed to `"Managed My Position"` and `"Managed My Position Value"` in `en.json`. German (`de.json`) and French (`fr.json`) translations kept `"My"` as a verbatim brand-name token — matching the physical label on Somfy RTS remotes sold in those markets — rather than translating it as the possessive pronoun.

## 📖 Cumulative scope (v2.22.0 series)

beta.7 is cumulative. Everything in [v2.22.0-beta.1] through [v2.22.0-beta.6] is included:

- **Nine-phase internal refactor (Phases A–H, beta.1/beta.2):** Coordinator responsibility distributed into `managers/`, `state/`, `pipeline/`, and `cover_types/`. Coordinator is ~640 lines lighter vs v2.21.5. Manual-override detection in `managers/cover_command/state_classifier.py`; cover-type-specific behaviour on `CoverTypePolicy` subclasses; cover-type literal branching eliminated. Net diff for the refactor: 62 files, +3,682 / −1,251 lines.

- **Post-settle cap grace — false venetian override fix ([#33], beta.2):** `DualAxisSequencer.is_in_suppression_with_cap` holds the cap bypass active for 5 seconds after a position command stamp, preventing KNX, Shelly 2PM, FGR223, and similar actuators from tripping false manual-override when `cover.state` settles before the tilt-walk burst finishes publishing.

- **Phase I — config-flow plumbing refactor (beta.3):** Collapsed duplicated schema reads, step routing, and options-step boilerplate into shared helpers, reducing `config_flow.py` by 168 lines.

- **Legacy climate step + optional-key cleanup (beta.3):** Removed `async_step_climate` and `CLIMATE_SCHEMA` dead code; extracted optional-key constants for `WEATHER_OVERRIDE`, `LIGHT_CLOUD`, and `TEMPERATURE_CLIMATE` schemas to prevent [#377]-class drift; fixed a latent omission where `CONF_CLOUDY_POSITION` was missing from its inline optional-key list.

- **"Is Sunny" sensor can now be cleared in the options flow ([#377], beta.3):** `CONF_BINARY_SENSOR` added to the optional-entities list so a blank submission removes the prior value instead of silently keeping it.

- **MODE2 tilt climate handler: sun-side awareness + `min_position` warning ([#373], beta.4):** The climate handler now picks the correct MODE2 hemisphere from the sign of gamma; `TiltPolicy.climate_tilt_percentage` is the single conversion helper for all four call sites. A ⚠️ warning surfaces in the config-flow summary when MODE2 is active and `min_position` ≥ 50.

- **Venetian tilt-axis command-grace suppression ([#33], beta.4):** `VenetianPolicy.secondary_axis_check` now carries a suppression callback that OR-composes the back-rotate window with a `_is_in_tilt_command_grace` helper, closing the last false-override vector on the tilt axis during a command-grace window.

- **Venetian publish-lag anchored to settle transition + startup grace ([#33], beta.5):** `VENETIAN_POSITION_SETTLE_STARTUP_GRACE_SECONDS = 6.0` blocks premature stall declaration on slow-start actuators (Somfy IO, 3–5 s motor startup delay); `VENETIAN_BACKROTATE_PUBLISH_LAG_SECONDS = 45.0` anchors the cap bypass to the observed `moving → settled` transition rather than to `stamp_position_command`, closing the last false-override vector on slow-bus republish paths.

- **Pipeline-aware My Position button and number ([#409], beta.6):** Two new entities per managed cover — `button.<managed_cover>_my_position` and `number.<managed_cover>_my_position_value` — expose the Somfy "My" preset through the managed entity, with pipeline preemption and min-mode floor clamping applied automatically.

- **Auto-control re-enable dispatches fresh solar position ([#352], beta.7, above):** Setting the automatic-control switch OFF→ON no longer sends the stale prior-cycle position before the pipeline runs; it sets `coordinator.state_change = True` and lets `async_refresh` route dispatch through the normal state-change path.

- **Custom sunrise/sunset time entity options ([#411], beta.7, above).**

- **My Position entity fixes ([#409], beta.7, above).**
