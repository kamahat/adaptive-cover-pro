# v2.22.1-beta.1

beta.1 adds two new fields to `sensor.<cover>_decision_trace` that let the companion Lovelace card tell whether a custom position floor is actively constraining the cover or just a no-op. Feedback welcome on the field semantics before they land in stable.

## 🎯 Should I install this?

- **If you use the companion Lovelace card and have custom position sensors configured in min-mode**, install this. The card can now distinguish a floor that's raising the cover above the autonomous calculation from one that's along for the ride.
- **If you are on v2.22.0 and it's working**, this update is low-risk — no existing state or attributes change. The two new fields appear only when a `CustomPositionHandler` wins the pipeline.
- **If you don't use custom position sensors or the companion card**, there is no user-visible change; stay on v2.22.0.

## ✨ Features

- **`active_slot` and `floor_binding` on the decision trace ([#421], PR [#422]):**

  `sensor.<cover>_decision_trace` gains two new attributes when a Custom Position handler wins the pipeline:

  **`active_slot`** — the 1-based slot number of the winning custom position handler (1–4). `None` when any other handler wins. Lets the card highlight which slot is in control without parsing the reason string.

  **`floor_binding`** — present only when `active_slot` is set and `min_mode` is active. `True` when the configured floor is raising the cover above the raw autonomous calculation (the floor is actively constraining). `False` when the raw calculation already meets or exceeds the floor (the floor is a no-op). `None` in exact mode (`min_mode=False`) or on the `use_my` path where min-mode semantics don't apply.

  Both fields are populated in `PipelineResult` by `CustomPositionHandler` and surfaced in `sensor.py`'s `_decision_trace_attrs` function. State and all existing attributes are unchanged; this is a purely additive surface.
